# SPEC — Εφαρμογή Αυτόματης Δημιουργίας Εβδομαδιαίου Προγράμματος Βαρδιών

## 1. Περιγραφή

Desktop-class web εφαρμογή για call center που διαχειρίζεται agents και παράγει αυτόματα το εβδομαδιαίο πρόγραμμα βαρδιών, με δυνατότητα χειροκίνητων διορθώσεων και εξαγωγή σε αρχείο Excel **πανομοιότυπο** με το υπάρχον χειροκίνητο format (μέρες σε οριζόντια μπλοκ, ωριαίες γραμμές 00:00–24:00, χρωματισμός κελιών ανά γραμμή/ρόλο).

Χρήστες: προϊστάμενοι / supervisors της ομάδας, μέσω browser.

## 2. Τεχνολογίες

| Κομμάτι | Επιλογή |
|---|---|
| Πλατφόρμα | Web εφαρμογή — Node.js + Express (backend), HTML/CSS/JS (frontend) |
| Βάση δεδομένων | MariaDB/MySQL μέσω `mysql2` |
| Authentication | Login προϊσταμένων (session-based, `express-session` + bcrypt για κωδικούς) |
| Excel export | `exceljs` server-side — ο χρήστης κατεβάζει το αρχείο από τον browser |
| Ανάπτυξη (φάση 1) | Localhost στο PC του developer: XAMPP MariaDB + `npm run dev` → `http://localhost:3000` |
| Παραγωγή (φάση 2) | Είτε PC/server εντός γραφείου (πρόσβαση από τοπικό δίκτυο, μηδέν κόστος) είτε VPS (π.χ. OVH) με PM2 + Nginx reverse proxy + HTTPS. Ο κώδικας ίδιος και στα δύο. |
| Γλώσσα UI | Ελληνικά |

## 3. Ρόλοι / Γραμμές (Roles)

Κάθε agent μπορεί να έχει έναν ή περισσότερους ρόλους. Κάθε ρόλος έχει χρώμα που χρησιμοποιείται στο Excel export (κωδικοί ARGB όπως στο υπάρχον αρχείο):

| Ρόλος | Χρώμα | ARGB |
|---|---|---|
| Supervisor | Μωβ | `FFC792D6` |
| Πειραιώς | Κίτρινο | `FFFFE699` |
| Ήρων | Πράσινο | `FFC6E0B4` |
| Eurobank | Μπλε | `FF66CCFF` |
| Υπόλοιπα call (εκτός Eurobank) | Πορτοκαλί | `FFF4B084` |
| Νέος agent | Μπλε κλειστό | `FF2F5496` (προς επιβεβαίωση) |
| Verification | Γκρι | `FFA6A6A6` |
| ??? (προς επιβεβαίωση) | Ροδακινί | `FFFCE4D6` |
| ??? (προς επιβεβαίωση) | Σκούρο κίτρινο | `FFFFC000` |

Σημείωση: Supervisor μπορούν να είναι ΜΟΝΟ όσοι έχουν τον ρόλο Supervisor (σήμερα: Παπασαράντου, Ψαρράς, Αμήρα, Αγγελούδη — αλλά αυτό ορίζεται δυναμικά από τη βάση, όχι hardcoded).

Οι ρόλοι/χρώματα να είναι επεξεργάσιμοι από οθόνη ρυθμίσεων (προσθήκη νέας γραμμής/πελάτη στο μέλλον χωρίς αλλαγή κώδικα).

## 4. Βάρδιες & Απαιτήσεις Κάλυψης

### 4.1 Καθημερινές (Δευτέρα–Παρασκευή)

| Βάρδια | Απαίτηση |
|---|---|
| 06:00–14:00 | 1 άτομο (Πειραιώς) |
| 07:00–15:00 | 1 Supervisor |
| 07:30–15:30 | 1 Eurobank + 1 International |
| 08:00–16:00 | 1 Eurobank (μόνο) + 1 Υπόλοιπα call + 1 Verification & call Eurobank + 1 Verification & call υπόλοιπα |
| 15:00–23:00 | 1 Supervisor |
| 15:30–23:30 | 1 Eurobank + 1 International |
| 16:00–24:00 | 1 Eurobank (μόνο) + 1 Υπόλοιπα call + 1 Verification & call Eurobank + 1 Verification & call υπόλοιπα |
| 19:00–03:00 | 1 Verification & call υπόλοιπα |
| 23:00–07:00 | 1 Eurobank (agent που κάνει βράδυ) |
| 23:30–07:30 | Εναλλακτική της νυχτερινής (βλ. κανόνα Κ4) |

### 4.2 Σαββατοκύριακα

| Βάρδια | Απαίτηση |
|---|---|
| 07:00–15:00 | 1 Supervisor |
| 07:30–15:30 | 1 Eurobank + 1 International |
| 08:00–16:00 | 1 Verification & call Eurobank + 1 Verification & call υπόλοιπα |
| 15:00–23:00 | 1 Supervisor |
| 15:30–23:30 | 1 Eurobank + 1 International |
| 16:00–24:00 | 1 Verification & call Eurobank + 1 Verification & call υπόλοιπα |
| 23:00–07:00 (ή 23:30–07:30) | 1 νυχτερινή Eurobank — τρέχει κανονικά και τα ΣΚ |

Πειραιώς/Ήρων τα ΣΚ: ΔΕΝ έχουν πλήρες σχήμα. Μπαίνουν μόνο agents με σταθερό ωράριο που καλύπτει ΣΚ.

Οι πίνακες κάλυψης να αποθηκεύονται στη βάση και να είναι επεξεργάσιμοι από οθόνη ρυθμίσεων (όχι hardcoded), ώστε αν αλλάξουν οι ανάγκες να αλλάζουν χωρίς νέο build.

## 5. Κανόνες (Constraints)

### Hard constraints (δεν παραβιάζονται ποτέ)

- **Κ1.** Κάθε απαίτηση κάλυψης του πίνακα 4 πρέπει να καλυφθεί από agent που έχει τον αντίστοιχο ρόλο.
- **Κ2.** Κάθε agent: το πολύ 1 βάρδια/ημέρα, 5 εργάσιμες + **2 ρεπό** την εβδομάδα.
- **Κ3.** Κανόνας Νικολιάδη (λόγω σχολής): κάθε Δευτέρα ΚΑΙ Παρασκευή έχει ΡΕΠΟ ή εργάζεται ΤΗΛΕΡΓΑΣΙΑ 06:00–14:00. Τα ΣΚ μόνο πρωινή βάρδια (εργαστήρια σχολής). Αντίστοιχος κανόνας για Νικολιάδη Αλίκη: κάθε Τρίτη ΚΑΙ Παρασκευή ρεπό ή τηλεργασία 06:00–14:00. Υλοποίηση ως γενικός μηχανισμός «περιορισμών ανά agent» (πίνακας agent_constraints) — ΟΛΕΣ οι ιδιαιτερότητες ανά agent (σταθερά ωράρια, μόνο πρωί/απόγευμα, τηλεργασία, σπαστά ωράρια, σταθερά ρεπό, soft προτιμήσεις) βρίσκονται στο seed_agents.json στο πεδίο constraints κάθε agent και πρέπει να τηρούνται από τον generator.
- **Κ4.** Νυχτερινή: αν το πρωί της επόμενης ανοίγει η Παπασαράντου (ή γενικά αν υπάρχει ήδη άτομο που ανοίγει στις 07:00/07:30), η νυχτερινή γίνεται 23:30–07:30 ώστε να μη χρειάζεται κάποιος στις 07:00 μόνο για άνοιγμα.
- **Κ5.** Σταθερά ωράρια: agents με flag «κλειδωμένο ωράριο» τοποθετούνται ΠΑΝΤΑ στο ωράριό τους (κυρίως Verification και Πειραιώς/Ήρων).
- **Κ6.** Άδειες/απουσίες: agent σε άδεια δεν προγραμματίζεται.
- **Κ7.** Νυχτερινές παίρνουν μόνο agents με flag «κάνει βράδυ».
- **Κ8.** Ελάχιστη ανάπαυση 11 ωρών μεταξύ δύο βαρδιών του ίδιου agent (νομοθεσία — ρυθμιζόμενο).
- **Κ9.** Βάρδια 19:00–03:00: την παίρνουν ΜΟΝΟ agents από τη λίστα επιλεξιμότητας, με μέγιστο όριο φορών/εβδομάδα και δηλωμένη τοποθεσία ο καθένας:

  | Agent | Μέγ./εβδομάδα | Τοποθεσία |
  |---|---|---|
  | ΠΑΝΤΕΛΙΔΗΣ | 1 | Γραφείο |
  | ΡΙΖΟΥ | 3 | Σπίτι (τηλεργασία) |
  | ΣΕΜΕΡΤΖΙΔΗΣ | 4 | Γραφείο |
  | ΜΠΟΥΓΙΟΥΚΟΣ | 2 | Γραφείο |
  | ΠΑΥΛΟΥ | 2 | Γραφείο |
  | ΝΤΟΥΦΑΣ | 4 | Γραφείο |
  | ΜΠΑΚΟΥΛΗΣ | 2 | Σπίτι (τηλεργασία) |
  | ΑΓΓΕΛΗ | 3 | Γραφείο — ΜΟΝΟ αν δεν είναι μόνη: πρέπει καθ' όλη τη διάρκεια 19:00–03:00 να υπάρχει τουλάχιστον ένα ακόμα άτομο στο γραφείο (π.χ. απογευματινές βάρδιες έως 24:00 + νυχτερινή από γραφείο) |

  **ΠΡΟΣΟΧΗ:** η λίστα αυτή ΥΠΕΡΙΣΧΥΕΙ των ατομικών περιορισμών ωραρίου — όποιος είναι στη λίστα μπορεί να πάρει τη 19:00–03:00 κανονικά (μέχρι το όριό του), ακόμα κι αν το προφίλ του λέει «μόνο πρωινή» ή «μόνο απογευματινή» βάρδια (π.χ. Παύλου).

  Η λίστα, τα όρια και οι τοποθεσίες αποθηκεύονται στη βάση (πίνακας `shift_eligibility`) και είναι επεξεργάσιμα από το UI — ο μηχανισμός να είναι γενικός ώστε να μπορεί να οριστεί αντίστοιχη λίστα και για άλλες βάρδιες (π.χ. νυχτερινή).

- **Κ10.** Απαγορεύεται συνεχόμενη 6ήμερη εργασία: κάθε agent μπορεί να δουλέψει το πολύ 5 συνεχόμενες ημέρες — μετά υποχρεωτικά ρεπό. Ο έλεγχος γίνεται ΚΑΙ στα σύνορα εβδομάδων (οι τελευταίες μέρες της προηγούμενης εβδομάδας μετράνε), συμπεριλαμβανομένης της τελευταίας αποθηκευμένης εβδομάδας ΠΡΙΝ την αρχή της περιόδου.

### Soft constraints (προτιμήσεις — ο αλγόριθμος προσπαθεί, με σειρά προτεραιότητας)

- **Σ1.** Παρασκευή 06:00–14:00: Νικολιάδης + Νικολιάδη Αλίκη μαζί για Πειραιώς (τηλεργασία, συμβατό με Κ3), αν βγαίνει το πρόγραμμα. Αν όχι, ο ένας από τους δύο παίρνει ρεπό εκείνη τη μέρα.
- **Σ2.** Τα 2 ρεπό κάθε agent να είναι συνεχόμενα, εκτός αν ο agent έχει ζητήσει συγκεκριμένες μέρες (αιτήματα ρεπό ανά εβδομάδα).
- **Σ3.** Δίκαιη κατανομή: εναλλαγή νυχτερινών και ΣΚ μεταξύ των agents που τα κάνουν (μετρητής ιστορικού στη βάση).
- **Σ4.** Συνέπεια: όπου γίνεται, ο ίδιος agent να κρατά παρόμοιο ωράριο μέσα στην εβδομάδα (όχι πρωί–βράδυ–πρωί).
- **Σ5.** Νικολιάδη Αλίκη: τις μέρες που δουλεύει κανονικά, ΜΟΝΟ πρωινή βάρδια με έναρξη 07:30 ή 08:00 (hard), και κατά προτίμηση με την ΙΔΙΑ ώρα έναρξης με τον Νικολιάδη Νίκο, επειδή έρχονται μαζί στη δουλειά (soft).

## 6. Δομή Βάσης Δεδομένων (MariaDB/MySQL)

Σημείωση σύνταξης: το παρακάτω σχήμα είναι ενδεικτικό σε γενική SQL μορφή — στην υλοποίηση χρησιμοποιούνται τύποι MariaDB (`INT AUTO_INCREMENT PRIMARY KEY`, `VARCHAR`/`TEXT`, `TINYINT(1)` για booleans, `JSON` όπου δηλώνεται JSON, `ENGINE=InnoDB`, `utf8mb4` για τα ελληνικά).

```sql
-- Χρήστες εφαρμογής (προϊστάμενοι) για login
CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  username TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,   -- bcrypt
  display_name TEXT,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- Ρόλοι/γραμμές με χρώμα για το Excel
CREATE TABLE roles (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,            -- π.χ. 'Eurobank'
  color_argb TEXT NOT NULL       -- π.χ. 'FF66CCFF'
);

-- Agents
CREATE TABLE agents (
  id INTEGER PRIMARY KEY,
  full_name TEXT NOT NULL,       -- όπως θα εμφανίζεται στο Excel
  company TEXT,                  -- 'OPTIVA' | 'INTELLI'
  departments TEXT NOT NULL,     -- JSON: ['call'] | ['verification'] | ['supervisor'] | συνδυασμός
  active INTEGER DEFAULT 1,      -- 0 = αποχώρησε (soft delete)
  can_night INTEGER DEFAULT 0,   -- κάνει νυχτερινές
  is_new INTEGER DEFAULT 0,      -- νέος agent (μπλε κλειστό)
  fixed_shift_start TEXT,        -- π.χ. '08:00' αν έχει κλειδωμένο ωράριο
  fixed_shift_end TEXT,          -- π.χ. '16:00'
  fixed_days TEXT,               -- JSON π.χ. '[1,2,3,4,5]' (1=Δευτέρα)
  notes TEXT                     -- ελεύθερες ιδιαιτερότητες
);

-- Ποιους ρόλους μπορεί να κάνει κάθε agent
CREATE TABLE agent_roles (
  agent_id INTEGER REFERENCES agents(id) ON DELETE CASCADE,
  role_id INTEGER REFERENCES roles(id) ON DELETE CASCADE,
  PRIMARY KEY (agent_id, role_id)
);

-- Ειδικοί κανόνες ανά agent (γενίκευση κανόνα Νικολιάδη)
CREATE TABLE agent_constraints (
  id INTEGER PRIMARY KEY,
  agent_id INTEGER REFERENCES agents(id) ON DELETE CASCADE,
  day_of_week INTEGER,           -- NULL = όλες οι μέρες
  allowed_shifts TEXT,           -- JSON λίστα επιτρεπτών βαρδιών
  required_shift TEXT,           -- υποχρεωτική βάρδια (π.χ. '06:00-14:00')
  description TEXT
);

-- Ορισμοί βαρδιών & απαιτήσεις κάλυψης (επεξεργάσιμα από ρυθμίσεις)
CREATE TABLE shift_requirements (
  id INTEGER PRIMARY KEY,
  day_type TEXT NOT NULL,        -- 'weekday' | 'weekend'
  start_time TEXT NOT NULL,
  end_time TEXT NOT NULL,
  role_id INTEGER REFERENCES roles(id),
  headcount INTEGER NOT NULL,
  label TEXT                     -- π.χ. 'Verification & call Eurobank'
);

-- Επιλεξιμότητα ειδικών βαρδιών (π.χ. 19:00-03:00) με όρια & τοποθεσία
CREATE TABLE shift_eligibility (
  id INTEGER PRIMARY KEY,
  agent_id INTEGER REFERENCES agents(id) ON DELETE CASCADE,
  shift_start TEXT NOT NULL,     -- '19:00'
  shift_end TEXT NOT NULL,       -- '03:00'
  max_per_week INTEGER NOT NULL, -- π.χ. 3
  location TEXT NOT NULL,        -- 'office' | 'home'
  not_alone INTEGER DEFAULT 0,   -- 1 = απαιτεί κι άλλο άτομο στο γραφείο σε όλη τη βάρδια
  notes TEXT
);

-- Αιτήματα ρεπό / άδειες ανά εβδομάδα
CREATE TABLE time_off (
  id INTEGER PRIMARY KEY,
  agent_id INTEGER REFERENCES agents(id) ON DELETE CASCADE,
  date TEXT NOT NULL,            -- 'YYYY-MM-DD'
  type TEXT NOT NULL,            -- 'repo_request' | 'leave' | 'sick'
  notes TEXT
);

-- Παραγόμενα προγράμματα (ιστορικό + μετρητές δικαιοσύνης)
CREATE TABLE schedules (
  id INTEGER PRIMARY KEY,
  week_start TEXT NOT NULL,      -- Δευτέρα της εβδομάδας
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  data TEXT NOT NULL             -- JSON: όλες οι αναθέσεις
);

CREATE TABLE assignments (
  id INTEGER PRIMARY KEY,
  schedule_id INTEGER REFERENCES schedules(id) ON DELETE CASCADE,
  agent_id INTEGER REFERENCES agents(id),
  date TEXT NOT NULL,
  start_time TEXT NOT NULL,
  end_time TEXT NOT NULL,
  role_id INTEGER REFERENCES roles(id),
  color_argb TEXT,               -- τελικό χρώμα ανάθεσης (επιλέγεται χειροκίνητα· default από τον ρόλο)
  is_manual_edit INTEGER DEFAULT 0
);
```

Σημείωση: για αποχώρηση agent προτιμάται `active = 0` αντί για DELETE, ώστε τα παλιά προγράμματα/ιστορικό να μη «σπάνε». Στο UI ο απενεργοποιημένος agent εξαφανίζεται από παντού.

**Δεξιότητες (skills):** εκτός από τους ρόλους/χρώματα, κάθε agent έχει και αναλυτικές δεξιότητες ανά project: `EUROBANK`, `ΑΠΕΔ`, `ALPHA`, `INTERNATIONAL`, `ΛΟΙΠΑ`, `ΗΡΩΝ`, `ΠΕΙΡΑΙΩΣ`. Ο πίνακας `agent_roles` γενικεύεται σε `agent_skills` με τα παραπάνω. Οι απαιτήσεις κάλυψης (πίνακας `shift_requirements`) δείχνουν σε skill, όχι σε χρώμα — το χρώμα του Excel προκύπτει από το skill της ανάθεσης.

**Αρχικά δεδομένα:** η εφαρμογή κάνει import το αρχείο `seed_agents.json` (παραδίδεται μαζί με το spec) την πρώτη εκκίνηση ή μέσω κουμπιού «Εισαγωγή agents». Περιέχει και τους 50 agents με εταιρία, τμήμα και δεξιότητες. Τα πεδία `can_night`, `fixed_shift`, `fixed_days` είναι `null` και συμπληρώνονται από τους προϊσταμένους μέσα στην εφαρμογή.

## 7. Οθόνες (UI)

0. **Login** — απλή φόρμα σύνδεσης προϊσταμένου (username/password). Διαχείριση χρηστών από την οθόνη Ρυθμίσεων.
1. **Agents** — λίστα με αναζήτηση. Προσθήκη/επεξεργασία: όνομα, ρόλοι (checkboxes με τα χρώματα), κάνει βράδυ, νέος agent, κλειδωμένο ωράριο (ώρες + μέρες), ειδικοί κανόνες, σημειώσεις. Κουμπί απενεργοποίησης (αποχώρηση).
2. **Περίοδος** — επιλογή μήνα ή ελεύθερου διαστήματος. Η εφαρμογή «στρογγυλοποιεί» αυτόματα σε πλήρεις εβδομάδες Δευτέρα–Κυριακή που καλύπτουν την περίοδο (π.χ. για Ιούλιο 2026 προτείνει 29/6–2/8) και δείχνει προεπισκόπηση των εβδομάδων που θα παραχθούν πριν την επιβεβαίωση. Καταχώρηση αδειών και αιτημάτων ρεπό ανά agent: επιλογή agent + τύπος (άδεια / αίτημα ρεπό / ασθένεια) + είτε μεμονωμένη ημερομηνία είτε ΔΙΑΣΤΗΜΑ ημερομηνιών (π.χ. άδεια 10/7–17/7 καταχωρείται μονομιάς, όχι μέρα-μέρα). Λίστα όλων των καταχωρήσεων της περιόδου με δυνατότητα διαγραφής. Στο preview grid οι μέρες άδειας/ρεπό-αιτήματος κάθε agent έχουν διακριτή ένδειξη ώστε να φαίνεται ΓΙΑΤΙ δεν προγραμματίστηκε κάποιος.
3. **Πρόγραμμα** — κουμπί «Δημιουργία Προγράμματος». Προεπισκόπηση σε grid ίδιας λογικής με το Excel (ώρες κάθετα, μέρες οριζόντια). **Το χρώμα κάθε ανάθεσης ΔΕΝ είναι δεμένο με τον agent** — ο generator απλώς προτείνει χρώμα βάσει της απαίτησης που καλύπτει η βάρδια, αλλά ο προϊστάμενος το αλλάζει ελεύθερα σε κάθε κελί/ανάθεση από dropdown με την παλέτα των ρόλων. Ό,τι χρώμα φαίνεται στο preview, αυτό γράφεται στο Excel. Χειροκίνητη διόρθωση αναθέσεων με drag & drop ή dropdown — οι αλλαγές ελέγχονται live για παραβιάσεις κανόνων και εμφανίζεται προειδοποίηση (όχι μπλοκάρισμα). Ένδειξη προβλημάτων: κόκκινο περίγραμμα σε ακάλυπτες απαιτήσεις.
4. **Ρυθμίσεις** — ρόλοι/χρώματα, πίνακες κάλυψης, θέση αρχείου βάσης, παράμετροι (ώρες ανάπαυσης κ.λπ.).

## 8. Αλγόριθμος Δημιουργίας

Προσέγγιση: **greedy με backtracking και scoring** (αρκετά για ~40 agents / εβδομάδα· αν αργότερα χρειαστεί βελτιστοποίηση, το interface του solver να είναι απομονωμένο ώστε να αντικατασταθεί π.χ. με CP-SAT μέσω service).

**Πολυεβδομαδιαία παραγωγή:** ο generator παράγει την περίοδο εβδομάδα-εβδομάδα με τη σειρά, μεταφέροντας κατάσταση από τη μία στην επόμενη: συνεχόμενες εργάσιμες ημέρες κάθε agent (για το Κ10), ώρα λήξης τελευταίας βάρδιας (για το 11ωρο Κ8), και μετρητές δικαιοσύνης νυχτερινών/ΣΚ (Σ3). Πριν την πρώτη εβδομάδα της περιόδου φορτώνει την τελευταία αποθηκευμένη εβδομάδα από τη βάση (αν υπάρχει) ώστε τα Κ8/Κ10 να ελέγχονται σωστά και στο ξεκίνημα.

Βήματα (ανά εβδομάδα):

1. **Κλείδωμα:** τοποθέτηση όλων των hard δεσμεύσεων — σταθερά ωράρια (Κ5), κανόνες agent (Κ3), άδειες (Κ6).
2. **Ρεπό:** κατανομή ρεπό — πρώτα τα αιτήματα, μετά συνεχόμενα ζεύγη (Σ2), με εξισορρόπηση ΣΚ (Σ3).
3. **Νυχτερινές:** ανάθεση σε agents με `can_night`, εναλλαγή βάσει ιστορικού (Σ3), επιλογή 23:00 ή 23:30 έναρξης βάσει Κ4.
4. **Κρίσιμες βάρδιες:** Supervisor, Eurobank, International — λίγοι υποψήφιοι, ανατίθενται πρώτοι.
5. **Υπόλοιπες βάρδιες:** για κάθε απαίτηση, βαθμολόγηση υποψηφίων (διαθεσιμότητα, δικαιοσύνη, συνέπεια ωραρίου Σ4) και επιλογή καλύτερου. Σε αδιέξοδο → backtracking.
6. **Soft κανόνας Σ1:** έλεγχος αν βγαίνει Νικολιάδης+Νικολιάδη Αλίκη Παρασκευή 06:00–14:00· αλλιώς ρεπό στον έναν.
7. **Αναφορά:** λίστα με ό,τι δεν καλύφθηκε ή παραβιάστηκε, για να το δει ο προϊστάμενος πριν το export.

## 9. Excel Export

- Ίδιο layout με το υπάρχον αρχείο: γραμμή 2 = ημέρες (`ΔΕΥΤΕΡΑ 6/7` … `ΚΥΡΙΑΚΗ 12/7`) ως μπλοκ στηλών, στήλη A = ωριαίες ζώνες `00:00-01:00` … `23:00-24:00`.
- Κάθε ανάθεση γράφεται στο μπλοκ της ημέρας, στις γραμμές των ωρών που καλύπτει, με **fill το χρώμα του ρόλου** της.
- Ετικέτες γραμμών (ΠΕΙΡΑΙΩΣ, ΗΡΩΝ, Verification/Call, INTERNATIONAL, ΓΡΑΦΕΙΟ, ΤΗΛΕΡΓΑΣΙΑ) όπου χρησιμοποιούνται στο υπάρχον format.
- Όνομα αρχείου: `Weekly_Program_DD_MM-DD_MM_YYYY.xlsx`.
- Εξαγωγή περιόδου: είτε ένα αρχείο ανά εβδομάδα, είτε ενιαίο workbook με ένα φύλλο ανά εβδομάδα (ονομασία φύλλου π.χ. `29.06-05.07`) — και τα δύο διαθέσιμα από το UI.
- Γραμματοσειρά/μέγεθος/πλάτη στηλών να αντιγραφούν από το υπάρχον αρχείο-δείγμα.

## 11. Φάσεις Ανάπτυξης

**Φάση 1 — Localhost (τώρα):**
1. XAMPP → δημιουργία βάσης `programa_vardion` (utf8mb4).
2. `npm init` project: Express server, σύνδεση MariaDB (`mysql2`), migrations για το σχήμα της ενότητας 6, import `seed_agents.json`.

4. Οθόνη Εβδομάδας (ρεπό/άδειες) + Generator (ενότητα 8).

7. Δοκιμή με πραγματικά δεδομένα 1-2 εβδομάδων από τους προϊσταμένους.

**Φάση 2 — Παραγωγή (όταν εγκριθεί):**
- Σενάριο Α: PC/server εντός γραφείου. Εγκατάσταση Node + MariaDB, εκκίνηση με PM2, πρόσβαση από τοπικό δίκτυο (`http://IP:3000`). Μηδέν μηνιαίο κόστος, δεδομένα εντός εταιρίας.
